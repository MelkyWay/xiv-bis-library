# Contributing

Thank you for helping keep XIV BiS Library accurate and useful.

## English

The project accepts four main kinds of contributions:

1. Suggest a new set when a new BiS, prog, or encounter-specific setup should be added.
2. Report a broken link when an existing XIVGear or Etro URL no longer works or points to the wrong set.
3. Report stale data when a set is still valid but needs to be updated after a patch, balance change, or guide revision.
4. Report an i18n issue when a translation, locale-specific label, or localized formatting issue needs attention.

### Preferred Path

For most contributors, the easiest path is to open one of the GitHub issue forms linked from the repository. That keeps submissions structured and lets me review them before anything changes in the live data file.

### What We Check

Before a submission is merged, I verify:

- the job and content category are correct
- the link works and points to the intended set
- the source is credible and current
- the entry does not duplicate something already present
- the data file still passes validation in CI

### What Happens Next

- If the report is actionable, I or a trusted contributor turns it into a JSON update or PR.
- If the report is missing details, it may be sent back for clarification.
- If the submission is already covered by an existing entry, it may be closed as duplicate or merged into a broader update.

### Discord

Discord is great for discussion and quick coordination, but the GitHub issue or PR is the source of record for the actual data change.

### Notes for Contributors

- If you know the exact source, include it.
- If you are suggesting a set, mention the encounter, job, and category clearly.
- If you are reporting a broken link, include the current URL and the replacement if you have one.
- If you are reporting stale data, explain what changed and when it changed.

### I18n Issues

Use the dedicated i18n issue form when a translation, locale-specific label, or localized formatting issue needs attention.

If you would like to help translate this guide into another language, contributions are welcome in any language.

## Français

Le projet accepte quatre grands types de contributions :

1. Proposer un nouveau set lorsqu’un BiS, un set de progression ou un ensemble spécifique à un combat doit être ajouté.
2. Signaler un lien cassé lorsqu’une URL XIVGear ou Etro ne fonctionne plus ou pointe vers le mauvais set.
3. Signaler une donnée obsolète lorsqu’un set reste valide mais doit être mis à jour après un patch, un changement d’équilibrage ou une révision du guide.
4. Signaler un problème i18n lorsqu’une traduction, un libellé localisé ou un format localisé doit être corrigé.

### Parcours recommandé

Pour la plupart des contributeurs, le plus simple est d’ouvrir l’un des formulaires GitHub liés depuis le dépôt. Cela garde les signalements structurés et me permet de les vérifier avant toute modification du fichier de données en production.

### Ce que nous vérifions

Avant qu’une contribution soit fusionnée, je vérifie :

- que le job et la catégorie de contenu sont corrects
- que le lien fonctionne et pointe vers le set attendu
- que la source est crédible et à jour
- que l’entrée ne duplique pas quelque chose qui existe déjà
- que le fichier de données passe toujours la validation dans CI

### Étapes suivantes

- Si le signalement est exploitable, je ou un contributeur de confiance le transforme en mise à jour JSON ou en PR.
- Si le signalement manque de détails, il peut être renvoyé pour clarification.
- Si le signalement est déjà couvert par une entrée existante, il peut être fermé comme doublon ou fusionné dans une mise à jour plus large.

### Discord

Discord est très utile pour les échanges et la coordination rapide, mais l’issue GitHub ou la PR reste la source de vérité pour la modification effective des données.

### Notes pour les contributeurs

- Si vous connaissez la source exacte, indiquez-la.
- Si vous proposez un set, précisez clairement le combat, le job et la catégorie.
- Si vous signalez un lien cassé, indiquez l’URL actuelle et, si vous l’avez, la nouvelle URL.
- Si vous signalez une donnée obsolète, expliquez ce qui a changé et quand cela a changé.

### Problèmes i18n

Utilisez le formulaire i18n dédié lorsqu’une traduction, un libellé localisé ou un format localisé doit être corrigé.

Si vous souhaitez aider à traduire ce guide dans une autre langue, les contributions sont les bienvenues dans n’importe quelle langue.
